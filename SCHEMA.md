# Supabase Database Schema

## Tables and Columns

### users
```sql
- id (uuid, primary key) - auto generated by Supabase Auth
- name (text, nullable)
- email (text, unique)
- avatar_url (text, nullable)
- role (text) - 'user' | 'business' | 'admin'
- verified (boolean) - default false
- created_at (timestamp) - default now()
- updated_at (timestamp) - default now()
```

### businesses
```sql
- id (uuid, primary key)
- owner_id (uuid, foreign key -> users.id)
- name (text)
- slug (text, unique) - for SEO URLs
- category (text) - e.g., 'PC Repair', 'WiFi & Internet'
- description (text)
- city (text)
- address (text)
- phone (text)
- whatsapp (text, nullable)
- email (text, nullable)
- website (text, nullable)
- logo_url (text, nullable) - Supabase Storage URL
- cover_url (text, nullable) - Supabase Storage URL
- gallery_urls (text[], nullable) - Array of storage URLs
- rating (numeric) - average rating
- reviews_count (integer) - default 0
- verified (boolean) - default false
- featured (boolean) - default false
- approved (boolean) - default false - MUST be true for public display
- price_range (text) - '$' | '$$' | '$$$'
- coordinates_lat (numeric, nullable)
- coordinates_lng (numeric, nullable)
- social_facebook (text, nullable)
- social_instagram (text, nullable)
- opening_hours (jsonb, nullable) - array of {day, hours}
- created_at (timestamp) - default now()
- updated_at (timestamp) - default now()
```

### reviews
```sql
- id (uuid, primary key)
- business_id (uuid, foreign key -> businesses.id)
- user_id (uuid, foreign key -> users.id)
- rating (integer) - 1-5
- comment (text)
- created_at (timestamp) - default now()
- updated_at (timestamp) - default now()
```

### favorites
```sql
- id (uuid, primary key)
- user_id (uuid, foreign key -> users.id)
- business_id (uuid, foreign key -> businesses.id)
- created_at (timestamp) - default now()
- UNIQUE(user_id, business_id)
```

## Storage Buckets

### business-logos
- Public: images/{business_id}/logo.{ext}

### business-covers
- Public: images/{business_id}/cover.{ext}

### business-gallery
- Public: images/{business_id}/gallery/{filename}

## Row Level Security (RLS) Policies

### users table
- Public: select only non-sensitive fields
- Private: users can update their own profile

### businesses table
- Public: select only approved AND verified businesses
- Owner: can see all their own businesses regardless of approval
- Admin: can see all businesses

### reviews table
- Public: select all
- Users: can insert/update/delete their own reviews

### favorites table
- Private: users can only view/modify their own favorites

## Admin User

To create an admin user manually:
```sql
INSERT INTO users (id, email, name, role, verified)
VALUES ('admin-uuid', 'admin@cityservices.com', 'Admin', 'admin', true);
```

## Key Business Logic

1. Businesses ONLY display if: `approved = true` AND `verified = true`
2. Business owners can see ALL their businesses (for dashboard)
3. Admins approve/reject businesses via admin dashboard
4. Users get verified badge if `verified = true`
5. Featured businesses sorted by rating and approval status
6. Search filters by category, city, rating
7. Email verification required for account activation
8. Unverified users have limited dashboard actions
